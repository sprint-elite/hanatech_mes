## 한 장 요약 — 공장에서 데이터가 흐르는 순서

```
[기준정보]          [계획·실행]              [현장]           [결과]
 품목·작업장    →   계획 → 작업지시 → LOT  →  실적 입력   →  재고·이력
 EBOM·MBOM              ↑                      │
 (레시피·공정)          사무실이 등록            작업자가 입력
```

**한 줄 메시지:** 사무실은 **「무엇을, 얼마나」** 를 정하고, 현장은 **「실제로 만든 수」** 만 넣으면, 재고와 이력이 **자동으로** 맞춰집니다.

### 0. 오프닝 (말할 내용)

- 화장품 **용기 조립·가공** 라인을 위한 **MES(생산실행시스템)** 입니다.  
  「**무엇을, 몇 개, 언제 만들었는지**」를 한곳에 기록·추적하는 시스템입니다.

- 이 프로그램은 **데이터가 이어지는 순서**로 구성되어 있습니다.

```
① 기준정보     품목 · 자재 레시피(EBOM) · 공정(MBOM)
      ↓
② 계획·지시    이번에 몇 개 만들지(계획) → 라인에 내릴 명령(작업지시)
      ↓
③ LOT          현장에서 추적할 작업 묶음 번호 발행
      ↓
④ 실적         실제로 만든 수량(양품·불량) 입력
      ↓
⑤ 결과         실적 이력 · 재고(완제품↑ 자재↓) · 불량 이력
```

- **오늘 시연**도 이 순서대로 갑니다.  
  대시보드로 **전체 상태**를 보고 → **기준정보**는 어떤 기준으로 돌아가는지 짧게 → **통합 생산 운영**에서 계획·지시·LOT → **현장 입력**에서 실적 → **재고·이력**으로 숫자가 맞는지 확인합니다.

- 앞 단계가 없으면 뒤가 막힙니다. (예: LOT 없으면 실적 불가, EBOM 없으면 자재 자동 차감 불가)  
  그래서 **한 줄로 이어지는 흐름**이 이 MES의 핵심입니다.

**말할 것 (예시 멘트)**

> 「안녕하세요. 오늘은 화장품 용기 조립 라인을 가정한 **HANA-TECH MES** 를 보여 드리겠습니다.  
> 엑셀·종이로 나눠 쓰던 정보를 **한 시스템**에 모아, **계획 → 지시 → LOT → 실적 → 재고**까지 이어지게 하는 게 목표입니다.  
> 예를 들어 완제품 **『화장품용기 세트 A』** 를 **100개** 만든다고 하면, 사무실에서 계획과 지시·LOT를 내리고, 현장에서는 **LOT 번호만 보고 수량을 입력**합니다. 그러면 재고가 자동으로 맞춰집니다.  
> 지금부터 그 **한 바퀴**를 화면으로 같이 보겠습니다.」

### 1. 대시보드

**메뉴:** 운영 → **대시보드**

**보여 줄 것**

- 상단 KPI — 지시 완료율, 불량률, 지연·주의
- 작업 상태(완료/진행/대기/보류)
- 최근 실적 차트, 작업지시·일정

**말할 것**

> 「먼저 **대시보드**입니다. 공장 전체를 한 화면에서 봅니다.  
> 예를 들어 지시 완료율이 **80%** 라고 나오면, 아직 끝나지 않은 지시가 있다는 뜻이고,  
> **지연·주의 2건**이 보이면 종료일이 지난 계획이나 지시가 있다는 신호입니다.  
> 아래 **작업 상태**에서 완료 5 · 진행 3 · 대기 2처럼 **지금 라인이 어떻게 돌아가는지** 숫자로 보입니다.  
> 오른쪽 **최근 7일 양품·불량** 차트는 품질 추이를 한눈에 보여 줍니다.  
> 실무에서는 아침에 여기부터 열어 보고, 이상 있으면 **통합 생산 운영**으로 들어갑니다.」

### 2. 기준정보 (훑어보기만)

**메뉴:** 기준정보 그룹 — **품목** → **EBOM** → **MBOM 공정** 순으로 짧게

| 화면 | 말할 것 (각 1분 내외) |
|------|----------------------|
| **품목** | 「**품목**은 모든 데이터의 시작입니다. 예: 완제품 **`CAP-BOX-01` 화장품용기 세트 A**, 원자재 **펌프캡·용기 본체**처럼 코드와 이름을 등록해 둡니다. 한 품목에 생산·구매·품질 정보가 같이 묶입니다.」 |
| **EBOM** | 「**EBOM**은 레시피입니다. 용기 세트 **1개**를 만들 때 펌프캡 **1개**, 본체 **1개**가 들어간다고 시스템에 적어 둡니다. 나중에 실적을 넣으면 이 기준으로 **자재가 자동 차감**됩니다.」 |
| **MBOM 공정** | 「**MBOM**은 만드는 **순서**입니다. 예: **`P10` 조립** 공정 하나만 있어도 됩니다. 현장 실적이 **어느 공정 실적인지** 여기서 판단합니다.」 |

**넘어가기 전 한 줄**

> 「정리하면, **품목 = 이름표**, **EBOM = 자재 레시피**, **MBOM = 공정 순서**입니다.  
> 이건 공장 세팅 때 **한 번** 등록해 두고, 오늘 본편은 **`PLAN-001` → `WO-001` → `LOT-001`** 로 이어지는 **실행 흐름**입니다.」

*(시간이 부족하면 품목 목록만 보여 주고 EBOM/MBOM은 말로만 설명해도 됩니다.)*

### 3. 통합 생산 운영 ★ 핵심

**메뉴:** 운영 → **통합 생산 운영**

**화면 구조를 먼저 설명**

> 「이 화면이 사무실에서 **가장 많이 쓰는 메인**입니다.  
> 위에 **생산계획 · 작업지시 · 생산 LOT · 재고현황** 탭이 있고, 왼쪽에서 품목·계획·지시를 고른 뒤 오른쪽에서 등록합니다.  
> 오늘은 **`CAP-BOX-01` 용기 세트**를 **100개** 만든다고 가정하고 보겠습니다.」

#### 3-1. 생산계획 탭

- 왼쪽에서 **`CAP-BOX-01`** 클릭 → 계획 목록
- **`PLAN-001`** 행 클릭 (없으면 등록 시연)

**말할 것 (예시)**

> 「**생산계획** 탭입니다. **`PLAN-001`**, 이번 주 **100개** 생산 계획이 잡혀 있습니다.  
> ‘이번에 **몇 개** 만들지’를 먼저 정하는 단계입니다.」

#### 3-2. 작업지시 탭

- 왼쪽 **`PLAN-001`** 선택 → **`WO-001`** 클릭

**말할 것 (예시)**

> 「다음 **작업지시**입니다. **`WO-001`**, 계획과 연결되어 **100개**, **`LINE-01` 1조립라인**에 배정, 작업자도 지정해 둡니다.  
> 상태가 **진행**이어야 현장에서 실적을 넣을 수 있습니다. **보류**로 두면 현장 화면에서 막힙니다 — 이건 의도된 통제입니다.」

#### 3-3. 생산 LOT 탭

- 왼쪽 **`WO-001`** 선택 → **잔여배정** · **`LOT-001`** 강조

**말할 것 (예시)**

> 「현장에 줄 번호가 **생산 LOT**입니다. **`LOT-001`**, 수량 **100개**로 발행했습니다.  
> 위에 **잔여배정 0**이면 지시 수량을 다 LOT로 나눈 상태입니다.  
> 만약 지시 100개인데 LOT를 60개만 만들었다면 잔여배정 **40**이 남고, 나머지 40짜리 LOT를 추가로 만들 수 있습니다.  
> 반대로 잔여가 **0**인데 또 등록하려 하면 **『LOT 등록 불가』** 안내가 뜹니다 — 시스템이 수량을 넘기지 않게 막습니다.」

#### 3-4. 재고현황 탭 (짧게)

**말할 것 (예시)**

> 「같은 화면 **재고현황** 탭에서 **`CAP-BOX-01`** 완제품이나 자재 잔량 요약을 볼 수 있습니다.  
> 실적 입력 **전후** 숫자가 어떻게 바뀌는지, 마지막에 다시 확인하겠습니다.」

### 4. 현장 실적 입력 ★ 임팩트

**새 탭:** http://3.35.9.204/worker-input  
*(또는 메뉴: 생산·LOT → 현장 입력)*

**보여 줄 것**

1. LOT 목록·검색
2. LOT 선택 → **투입 / 양품 / 불량**
3. (불량 있으면) 유형별 입력 — 없으면 자동 건너뜀 설명
4. **실적 저장** → 완료 메시지

**말할 것 (예시)**

> 「이제 **현장**입니다. 작업자는 이 주소만 씁니다. 메뉴가 거의 없습니다.  
> 오늘 할 일은 **`LOT-001`** 을 찾는 것입니다. 검색창에 `LOT-001` 입력하거나 카드를 탭합니다.  
> 선택되면 노란색으로 표시됩니다. **다음** → 투입 **100**, 양품 **98**, 불량 **2** 입력 → **다음** → 불량 유형 **스크래치 2개** → **실적 저장**.  
> 불량이 **0**이면 ③ 단계는 자동으로 건너뜁니다.  
> 「**LOT 고르고, 숫자 넣고, 저장**」 — 현장 교육은 이 한 문장이면 됩니다.  
> 사무실에서 이미 내려준 **`WO-001` / `LOT-001`** 만 알면 됩니다.」

**시연 팁:** 미리 등록한 LOT로 **저장까지** 한 번 보여 주면 흐름이 완결됩니다. (이미 완료된 LOT면 「이 LOT는 아까 반영한 건이라, 새 LOT로 다시 보여 드리겠습니다」라고 말한 뒤 다른 LOT 사용)

### 5. 결과 확인 — 숫자가 맞는지

**메인 탭으로 복귀**

| 순서 | 메뉴 | 말할 것 (예시) |
|------|------|----------------|
| 1 | 생산·LOT → **실적 이력** | 「방금 **`LOT-001`** 에 넣은 **양품 98 · 불량 2** 가 여기 남습니다. 언제, 어떤 공정에서 넣었는지 추적됩니다.」 |
| 2 | 재고·출하 → **재고** | 「**`CAP-BOX-01`** 완제품 재고가 **98개 늘었**고, EBOM대로 **펌프캡·본체 자재**는 **98세트만큼 줄었**습니다. 사람이 따로 출고 입력 안 해도 **백플러시**로 맞춰집니다.」 |
| 3 | (선택) **자재 투입** / **불량이력** | 「**자재 투입**에서 어떤 LOT에 어떤 자재가 쓰였는지, **불량이력**에서 스크래치 2건을 볼 수 있습니다.」 |

**한 줄**

> 「정리하면 **`PLAN-001` → `WO-001` → `LOT-001` → 현장 98/2** 까지 한 사이클이고, 재고·이력이 **자동 반영**된 상태입니다.」

### 7. 마무리 (말할 내용)

**말할 것 (예시)**

> 「오늘 보신 흐름을 다시 말씀드리면,  
> **기준정보**(품목·EBOM·MBOM) → **계획 `PLAN-001`** → **지시 `WO-001`** → **LOT `LOT-001`** → **현장 실적** → **재고·이력** 입니다.

> 자세한 사용법은 **사용설명서**, 현장 1페이지는 **WORKER-QUICK** 으로 드리겠습니다. 질문 받겠습니다.」

| 현장에서 뭘 등록하나요? | 「**실적 숫자만**입니다. `LOT-001` 고르고 98·2 넣고 저장 — 끝입니다.」 |
| 공정이 여러 개면? | 「공정이 `P10` 조립, `P20` 검사처럼 여러 개면 **실적 등록** 메뉴에서 공정을 고릅니다. 오늘은 조립 1공정 예시였습니다.」 |
| 자재는 언제 빠지나요? | 「현장에서 **실적 저장**하는 순간, EBOM에 적어 둔 **펌프캡 1 + 본체 1** 이 양품 98만큼 **자동 출고**됩니다.」 |
| 바코드·출하도 되나요? | 「메뉴에 **바코드·출하·외주**까지 있습니다. 오늘은 **생산 한 바퀴**만 보여 드렸습니다.」 |
| 핸드폰만 되나요? | 「**worker-input** 은 태블릿·핸드폰 화면에 맞춰져 있습니다.」 |
| 어두운 화면 싫어요 | 「왼쪽 **밝은 모드** 버튼으로 흰 배경으로 바꿀 수 있습니다.」 |
