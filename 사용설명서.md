# HANA-TECH MES 사용설명서

> **개발·IT 지식이 없어도** 따라 할 수 있도록 쓴 안내입니다.  
> 공장 사무실·현장·라인에서 **매일 실제로 쓰는 방법**만 정리했습니다.

---

## 1. 먼저 읽어 주세요

### 이 프로그램은 무엇인가요?

**MES**는 공장에서 **「무엇을, 몇 개, 언제 만들었는지」** 를 컴퓨터에 기록하는 시스템입니다.

- **사무실** — 품목·계획·작업지시·LOT 를 미리 등록합니다.
- **현장** — 실제로 만든 수량(양품·불량)만 입력합니다.
- **라인장** — 큰 화면으로 라인 상태를 봅니다.

### 접속 주소 (북마크해 두세요)

| 누가 쓰나요 | 주소 | 비고 |
|-------------|------|------|
| **사무실·관리** | http://3.35.9.204 | 왼쪽에 메뉴가 보이는 화면 |
| **현장 작업자** | http://3.35.9.204/worker-input | **핸드폰·태블릿** 에서 쓰기 좋음 |
| **라인 전광판** | http://3.35.9.204/floor-board | 모니터에 띄워 두는 화면 |

### 브라우저

- **크롬** 또는 **엣지** 를 쓰세요.
- 화면이 예전 그대로면: 키보드 **Ctrl + Shift + R** (맥은 **Cmd + Shift + R**) 로 **새로고침**.

### 화면이 너무 어두우면

왼쪽 메뉴 **맨 위** 에 **「밝은 모드」** 버튼이 있습니다. 누르면 흰 배경으로 바뀝니다.

## 2. 화면 익히기 (30초)

사무실 화면은 대략 이렇게 생겼습니다.

```
┌──────────────┬────────────────────────────────────┐
│  HANA-TECH   │  (오른쪽) 오늘 할 일                 │
│  MES         │                                    │
│              │  제목 · 표 · 등록 버튼 …             │
│  [밝은 모드]  │                                    │
│              │                                    │
│  ▼ 운영      │                                    │
│  ▼ 기준정보   │                                    │
│  ▼ 계획·지시  │                                    │
│  ▼ 생산·LOT  │                                    │
│  …           │                                    │
└──────────────┴────────────────────────────────────┘
```

| 부분 | 하는 일 |
|------|---------|
| **왼쪽 메뉴** | 글자를 누르면 아래 항목이 펼쳐집니다. 이동할 메뉴를 **한 번 더** 누르세요. |
| **오른쪽** | 선택한 메뉴의 목록·등록 창이 나옵니다. |
| **신규 등록 / 새 ○○** | 새 데이터를 만듭니다. |
| **저장** | 입력한 내용을 확정합니다. **반드시 눌러야** 반영됩니다. |
| **취소 / 닫기** | 저장하지 않고 창을 닫습니다. |

> **팁:** 빨간 글씨나 안내 창이 뜨면, 보통 **필수 칸이 비었거나** 숫자가 맞지 않을 때입니다. 메시지를 읽고 해당 칸을 고치세요.

## 3. 역할별 — 하루에 뭘 하나요?

### 3-1. 현장 작업자 (조립·가공 라인)

**할 일은 하나입니다. → 실적(만든 개수) 입력**

1. **http://3.35.9.204/worker-input** 접속 (즐겨찾기 권장)
2. 사무실에서 알려준 **LOT 번호** 로 LOT 선택
3. **투입 · 양품 · 불량** 수량 입력
4. **실적 저장**

자세한 순서는 [5장. 현장 실적 입력](#5-현장-실적-입력-작업자용) 을 보세요.

**작업자가 하지 않는 것:** 품목 등록, 계획, 작업지시, BOM — 전부 **사무실** 담당입니다. 막히면 사무실에 연락하세요.

---

### 3-2. 사무실 생산관리 (매일)

이미 품목·BOM 등 **기초 등록이 끝난 뒤**, 매일 아침·저녁에 아래만 하면 됩니다.

#### 아침 (약 10분)

| 순서 | 메뉴 | 할 일 |
|------|------|--------|
| 1 | **운영** → **대시보드** | 지시 완료율, 지연·주의 건수 확인 |
| 2 | **운영** → **통합 생산 운영** | **작업지시** 탭 — 오늘 지시가 **대기** 또는 **진행** 인지 확인 |
| 3 | 같은 화면 **생산 LOT** 탭 | 현장에 줄 LOT 가 부족하면 **LOT 등록** |

**주의:** 작업지시 상태가 **보류(HOLD)** 이면 현장에서 실적을 **넣을 수 없습니다.** 보류면 **진행** 으로 바꿔 주세요.

#### 낮 (필요할 때만)

- 계획 수량 변경, LOT 추가 분할
- 자재 부족 시 **자재 LOT** 입고

#### 퇴근 전 (약 10분)

| 메뉴 | 확인 내용 |
|------|-----------|
| **생산·LOT** → **실적 이력** | 오늘 입력된 양품·불량 |
| **재고·출하·외주** → **재고** | 완제품·자재 수량이 맞는지 |

## 4. 사무실 — 자주 쓰는 메뉴

### 4-1. 통합 생산 운영 (가장 많이 씀)

**메뉴:** 왼쪽 **운영** → **통합 생산 운영**

한 화면에서 **계획 → 지시 → LOT → 재고** 를 탭으로 바꿔 가며 처리합니다.

```
[ 탭 ]  생산계획 | 작업지시 | 생산 LOT | 재고현황

[ 왼쪽 ]  품목·계획·지시 목록  ← 여기서 먼저 클릭

[ 오른쪽 ]  등록 버튼 · 표
```

| 탭 | 언제 쓰나요 |
|----|-------------|
| **생산계획** | 이번 주·이번 달 **몇 개 만들지** 계획 |
| **작업지시** | 계획을 실제 라인 **지시**로 내림 (라인·작업자 배정) |
| **생산 LOT** | 지시를 **현장에서 쓸 LOT** 단위로 나눔 |
| **재고현황** | 품목별 재고 요약 |

#### 작업지시 등록 (요약)

1. **작업지시** 탭
2. 왼쪽에서 **계획** 선택
3. **지시 등록** 클릭
4. 지시번호, 수량, **배정 라인**, **작업자** 체크, 상태 **대기** 또는 **진행**
5. **저장**

#### 생산 LOT 등록 (요약)

1. **생산 LOT** 탭
2. 왼쪽에서 **작업지시** 선택
3. 위쪽 **잔여배정** 숫자 확인 (0 이면 더 이상 LOT 를 못 만듦)
4. **LOT 등록** 클릭 → LOT 번호·수량 입력 → **저장**
5. 현장에 **LOT 번호** 를 알려 줌

> **잔여배정이 0** 인데 LOT 등록을 누르면 **「LOT 등록 불가」** 안내가 뜹니다. 지시 수량을 늘리거나, 이미 만든 LOT 수량을 조정하세요.

---

### 4-2. 품목 등록

**메뉴:** **기준정보** → **품목** → **새 품목**

창이 열리면 **6개 블록** 이 있습니다. 처음에는 **기본정보** 만 채워도 됩니다.

| 블록 | 보통 언제 채우나요 |
|------|-------------------|
| **기본정보** | 항상 (코드, 이름, 완제품/원자재 구분) |
| 포장/물류 | 출하·단위가 필요할 때 |
| 구매/자재 | 사급 자재일 때 |
| 품질 | 검사 기준이 있을 때 |
| 재고/물류 | LOT 관리 품목일 때 |
| 외주 | 외주 공정일 때 |

**기본정보 필수 칸**

| 칸 | 설명 |
|----|------|
| 품목코드 | 공장에서 쓰는 고유 번호 (예: `CAP-BOX-01`) |
| 품목명 | 사람이 읽는 이름 |
| 품목 유형 | 완제품 / 반제품 / 원자재 |
| 생산 가능 | 완제품은 **Y**, 원자재만 사는 것은 **N** |

맨 위 **저장** 을 눌러야 목록에 나타납니다.

---

### 4-3. 자재 LOT (원자재 입고)

**메뉴:** **생산·LOT** → **자재 LOT**

| 버튼 | 하는 일 |
|------|---------|
| **신규 등록** | 입고한 자재 LOT · 수량 · 입고일 등록 |
| **재고 반영 IN** | 재고 숫자와 연결 (담당자 판단) |

왼쪽부터 순서대로: **신규 등록** → **상태 필터** → **검색** 이 붙어 있습니다.

---

### 4-4. 실적·재고 확인 (조회만)

| 메뉴 | 보는 것 |
|------|---------|
| **실적 이력** | 언제, 어떤 LOT 에 얼마나 넣었는지 |
| **자재 투입** | 생산에 쓰인 자재 |
| **재고** | 품목별 현재 수량 |
| **불량이력** | 불량 유형·수량 |

## 5. 현장 실적 입력 (작업자용)

**주소:** http://3.35.9.204/worker-input

### 준비물

- 사무실에서 받은 **LOT 번호** (예: `LOT-001`)
- 오늘 작업한 **개수**

### 입력 순서 (4단계)

```
① LOT 선택  →  ② 수량 입력  →  ③ 불량 상세  →  ④ 확인·저장
```

#### ① LOT 선택

1. 주소 접속
2. 검색창에 LOT 번호 입력 **또는** 목록에서 카드 탭
3. 선택된 카드가 **노란색** 으로 바뀌면 하단 **다음**

#### ② 수량 입력

| 칸 | 의미 |
|----|------|
| **투입** | 이번에 작업한 수량 |
| **양품** | 정상 제품 |
| **불량** | 불량 (없으면 **0**) |

**규칙:** 양품 + 불량 은 **투입보다 클 수 없습니다.**

- **「잔여 수량 한 번에 입력」** 버튼이 보이면, LOT 에 남은 수량을 한 번에 넣을 수 있습니다.
- 입력 후 **다음**

#### ③ 불량 상세

- 불량이 **0** 이면 이 단계는 **자동으로 건너뜁니다.**
- 불량이 있으면: 유형별 수량 입력 (합계 = 불량 수)
- **다음**

#### ④ 확인·저장

1. 숫자 다시 확인
2. **실적 저장**
3. **「실적이 저장되었습니다」** 가 나오면 완료
4. **다음 실적 등록** 으로 이어서 입력 가능

### 현장에서 자주 보는 메시지

| 화면 메시지 | 뜻 | 할 일 |
|-------------|-----|--------|
| 진행 중인 LOT가 없습니다 | 입력할 LOT 가 없음 | **사무실**에 LOT 발행 요청 |
| 보류(HOLD) 작업지시 | 지시가 멈춘 상태 | **사무실**에 지시 해제 요청 |
| 등록된 공정이 없습니다 | 공정이 안 잡혀 있음 | **사무실**에 MBOM 등록 요청 |
| 불량유형이 없습니다 | 불량 코드 없음 | **사무실**에 불량유형 등록 요청 |
| 자재 부족 | 원자재가 모자람 | **사무실**에 자재 입고 요청 |

## 6. 처음 한 번만 — 사무실 세팅 순서

공장에 **처음 MES 를 도입**할 때, 사무실에서 **한 번만** 아래 순서로 등록합니다.  
(이미 등록되어 있으면 **7장 매일 업무**만 보시면 됩니다.)

```
① 품목  →  ② 작업장·작업자  →  ③ MBOM 공정  →  ④ EBOM(자재 레시피)
    →  ⑤ 자재 LOT 입고  →  ⑥ 계획·지시·LOT  →  ⑦ 불량유형(불량 쓸 때)
```

| 단계 | 메뉴 | 한 줄 설명 |
|------|------|------------|
| ① | 기준정보 → **품목** | 완제품 1종 + 쓰는 원자재 등록 |
| ② | **작업장**, **작업자** | 라인·작업자 이름 등록 |
| ③ | **MBOM 공정** | 제품 만드는 **공정 순서** (처음엔 1개만 해도 됨) |
| ④ | **EBOM** | 완제품 1개당 **들어가는 자재·수량** |
| ⑤ | **자재 LOT** | 입고한 원자재 LOT·수량 |
| ⑥ | **통합 생산 운영** | 계획 → 지시 → 생산 LOT |
| ⑦ | **불량유형** | 불량 코드 (불량 0 이면 나중에 해도 됨) |

**연습용 예시 데이터**로 처음부터 따라 해 보고 싶으면, 기술 담당자용 문서 **[GUIDE.md](./GUIDE.md)** 의 「연습 시나리오」를 참고하세요. (코드 예: `CAP-BOX-01`, `LOT-001` 등)

## 7. 쉬운 용어 설명

| 용어 | 쉬운 말 |
|------|---------|
| **품목** | 제품·부품·자재의 **이름표** |
| **EBOM** | 완제품 1개 만들 때 들어가는 **자재 목록(레시피)** |
| **MBOM** | 제품을 만드는 **공정 순서** (조립 → 검사 …) |
| **생산 계획** | 이번 기간 **몇 개 만들 계획**인지 |
| **작업지시** | 라인에 내리는 **실제 작업 명령** |
| **생산 LOT** | 현장에서 추적하는 **작업 묶음 번호** |
| **자재 LOT** | 입고된 **원자재 묶음** |
| **잔여배정** | 지시 수량 중 아직 LOT 로 안 나눈 **남은 수량** |
| **실적** | 실제로 **만든 개수** (양품·불량) |
| **백플러시** | 실적 저장 시 쓴 자재를 **시스템이 자동으로 빼는 것** |

## 8. 문제 해결 — 이렇게 하세요

| 상황 | 확인·조치 |
|------|-----------|
| 저장이 안 됨 | 빨간 안내 읽기 → **필수 칸** 채우기 → 숫자 칸에 글자 없는지 확인 |
| 현장에서 LOT 가 안 보임 | 사무실: **생산 LOT** 등록했는지, 지시가 **보류** 아닌지 |
| LOT 등록 불가 | **잔여배정 0** — 지시 수량 늘리거나 기존 LOT 수정 |
| 실적 넣었는데 재고 안 맞음 | **EBOM** 자재 맞는지, **자재 LOT** 잔량 있는지 확인 |
| 공정 여러 개인 제품 | 현장 대신 **실적 등록** 메뉴에서 공정 직접 선택 |
| 화면이 깨지거나 옛날 화면 | **Ctrl + Shift + R** 새로고침 |
| 접속이 안 됨 | Wi‑Fi·공장 네트워크 확인 → **IT 담당자** |

**현장 작업자**는 위 표에서 해결이 안 되면 **사무실 생산관리**에게 연락하는 것이 원칙입니다.
