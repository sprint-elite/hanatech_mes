# 연습 시나리오 (처음 따라하기)

| 구분 | 코드 | 이름 |
|------|------|------|
| 완제품 | `CAP-BOX-01` | 화장품용기 세트 A |
| 원자재 | `RAW-CAP-01` | 펌프캡 |
| 원자재 | `RAW-BODY-01` | 용기 본체 |

## 1단계. 품목 3개 등록

**메뉴 경로:** 왼쪽 **기준정보** 클릭 → **품목** 클릭  
→ 참고: [품목 화면 설명](#i-1-품목)

### 공통 조작 (품목 1개마다 반복)

1. 오른쪽 위 **「새 품목」** 클릭  
2. **신규 등록** 창이 열림 (위에 **저장 · 취소 · 닫기** 버튼)  
3. **기본정보** 블록만 채움 (아래 표). **포장/물류 ~ 외주** 블록은 **스크롤만 내려 확인하고 건드리지 않아도 됨**  
4. 맨 위 **저장** 클릭 → 창 닫힘  
5. 목록에 방금 코드가 보이면 성공  

```
[ 신규 등록 창 구조 ]

  ┌─ 기본정보 ──────── ← 여기만 채움 (연습)
  ├─ 포장/물류 ─────── ← 기본값 그대로
  ├─ 구매/자재 ─────── ← 기본값 그대로
  ├─ 품질 ──────────── ← 기본값 그대로
  ├─ 재고/물류 ─────── ← 기본값 그대로
  └─ 외주 ──────────── ← 기본값 그대로
```

### 1-1. 펌프캡 (원자재)

| 칸 (기본정보 블록) | 입력 |
|--------------------|------|
| 품목코드 | `RAW-CAP-01` |
| 품목명 | `펌프캡` |
| 품목 유형 | **원자재** |
| 생산 가능 | **N** |

### 1-2. 용기 본체 (원자재)

| 품목코드 | 품목명 | 유형 | 생산 가능 |
|----------|--------|------|-----------|
| `RAW-BODY-01` | 용기 본체 | 원자재 | N |

### 1-3. 용기 세트 (완제품)

| 품목코드 | 품목명 | 유형 | 생산 가능 |
|----------|--------|------|-----------|
| `CAP-BOX-01` | 화장품용기 세트 A | 완제품 | Y |

### ✅ 확인 + 품목 ID 메모

1. 목록에 **3건** 보임  
2. **펌프캡** 행 클릭 → 창 제목에 `수정 (ID 1)` 처럼 **숫자** 표시 → **메모** (자재 LOT용)  
3. **용기 본체** ID도 같은 방법으로 메모

→ 참고: [MBOM 공정](#i-4-mbom-공정)

| 공정코드 | `P10` |
| 공정명 | `조립` |
| 작업장 | `LINE-01 — 1조립라인` |
| 외주 | N |
| 사용 | **Y** |

3. **「공정 추가」** 클릭  
4. 아래 표에 한 줄 생기면 성공

## 4단계. EBOM (자재 레시피)

→ 참고: [EBOM](#i-5-ebom)

1. **기준정보** → **EBOM**  
2. **BOM 선택** 드롭다운에서 `CAP-BOX-01` 선택  
3. **하위 BOM 추가/수정** 패널에서:

**펌프캡**

| 칸 | 입력 |
|----|------|
| 하위 품목 | `RAW-CAP-01 · 펌프캡` |
| 수량 | `1` |

→ **「하위 BOM 추가」** 클릭  

**용기 본체** — 같은 방법, 수량 `1` → **「하위 BOM 추가」**

4. 아래 표에 **2줄** 보이면 성공

## 5단계. 자재 LOT 입고

→ 참고: [자재 LOT](#i-7-자재-lot)

1. **생산·LOT** → **자재 LOT**  
2. **「신규 등록」** 클릭  

**펌프캡**

| 칸 | 입력 |
|----|------|
| LOT 번호 | `MAT-CAP-001` |
| 품목 ID | 1단계에서 메모한 **펌프캡 숫자** |
| 입고수량 | `1000` |
| 입고일 | 오늘 날짜 |
| 상태 | AVAILABLE (그대로) |

3. **저장** → **용기 본체**도 반복 (`MAT-BODY-001`, 본체 품목 ID)  
4. 목록 **잔량 1000** 확인  

> 실적 후 재고가 안 맞으면 **「재고 반영 IN」** 으로 자재 LOT ID·수량 입력 (사무실 담당자 판단).

## 6단계. 계획 → 지시 → LOT

→ 참고: [통합 생산 운영](#i-8-통합-생산-운영)

1. **운영** → **통합 생산 운영**  
2. 화면 구조:

```
[ 상단 탭 ]  생산계획 | 작업지시 | 생산 LOT | 재고현황
[ 요약 바 ]  제품명, 잔여배정 …
[ 왼쪽 ]     품목 / 계획 / 지시 목록  ← 여기서 선택
[ 오른쪽 ]   등록·표
```

### 6-1. 생산계획

1. 상단 **「생산계획」** 탭 클릭  
2. **왼쪽** 품목 목록에서 `CAP-BOX-01` 클릭  
3. 오른쪽 **「계획 등록」** 클릭  
4. 입력 후 **저장**:

| 칸 | 입력 |
|----|------|
| 계획번호 | `PLAN-001` |
| 품목 | `CAP-BOX-01` |
| 계획수량 | `100` |
| 상태 | 계획 또는 확정 |
| 시작일·종료일 | 이번 주 |

### 6-2. 작업지시

1. **「작업지시」** 탭 클릭  
2. **왼쪽**에서 `PLAN-001` 클릭  
3. **「지시 등록」** 클릭  
4. 입력 후 **저장**:

| 칸 | 입력 |
|----|------|
| 지시번호 | `WO-001` |
| 계획 | `PLAN-001` |
| 지시수량 | `100` |
| 배정 라인 | `LINE-01` |
| 상태 | **대기** 또는 **진행** (⚠️ 보류 금지) |
| 배정 작업자 | `홍길동` 체크 |

### 6-3. 생산 LOT

1. **「생산 LOT」** 탭 클릭  
2. **왼쪽**에서 `WO-001` 클릭  
3. 요약 바 **잔여배정**이 **100**인지 확인  
4. **「LOT 등록」** 클릭  

| 칸 | 입력 |
|----|------|
| LOT번호 | `LOT-001` |
| 작업지시 | `WO-001` |
| LOT수량 | `100` |

5. **저장**  
6. 제목 옆 `(배정 100/100, 잔여 0)` 확인  

> 잔여 0인데 LOT 등록 클릭 → **「LOT 등록 불가」** 안내 (정상). 지시수량 늘리거나 기존 LOT 수정.

## I-1. 품목

**메뉴:** `기준정보` → **품목** · **새 품목**

### 역할

모든 생산·재고·BOM의 **기준**. 품목 하나에 6개 프로필이 함께 저장됩니다.

### 기본정보

| 필드 | 구분 | 기능 |
|------|------|------|
| 품목코드 | **필수** | 고유 코드 |
| 품목명 | **필수** | 표시 이름 |
| 품번 | 선택 | 사내 품번 |
| 품목 유형 | **필수** | 완제품 / 반제품 / 원자재 |
| 생산 가능 | **필수** | Y=생산 대상, N=자재 등 |
| 상태 | 기본 ACTIVE | ACTIVE / INACTIVE |

### 포장/물류

| 필드 | 구분 | 기능 |
|------|------|------|
| 기본 단위 | 기본 EA | EA, BOX 등 |
| 표준 포장 수량 | 선택 | 1박스 입수 |
| 바코드 | 선택 | |
| 단위 중량 | 선택 | |
| 규격 | 선택 | |
| 안전재고 / 최대재고 | 선택 | 재고·통합운영 요약 |

### 구매/자재 · 품질 · 재고/물류 · 외주

| 블록 | 주요 필드 | 비고 |
|------|-----------|------|
| 구매/자재 | 구매 가능, 공급업체, 단가, MOQ | 기본 구매 가능 N |
| 품질 | 검사 필수, 검사 방식, 허용 불량률 | |
| 재고/물류 | LOT 관리, 매입처 | |
| 외주 | 외주 여부, 외주업체 | |

**저장 검증:** 숫자 칸은 비우거나 올바른 숫자만.

## I-4. MBOM 공정

**메뉴:** `기준정보` → **MBOM 공정**

**공정 순서** 정의. 실적·백플러시 판단에 사용.

| 필드 | 구분 | 기능 |
|------|------|------|
| 품목 | **필수** | |
| 순서 | **필수** | 10, 20, 30… |
| 공정코드 / 공정명 | **필수** | |
| 작업장 | 권장 | |
| 사용 | **필수** | **N이면 실적 불가** |

## I-5. EBOM

**메뉴:** `기준정보` → **EBOM**

완제품 1개당 **투입 자재 목록**. 백플러시에 사용.

상단: BOM 검색 · BOM 선택  
하위 패널: 하위 품목, 수량(**필수**), 단위, 사용여부 등

## I-6. 불량유형

**메뉴:** `기준정보` → **불량유형**

품목별 불량 코드. 현장 ③·불량이력에 사용.

## I-7. 자재 LOT

**메뉴:** `생산·LOT` → **자재 LOT**

원자재 입고 LOT·잔량. 생산 완료 시 차감.

| 필드 | 구분 |
|------|------|
| LOT 번호, 품목 ID, 입고수량, 입고일 | **필수** |
| 잔량, 공급사, 상태 | 선택/기본 |

**재고 반영 IN:** 재고 행·입고 트랜잭션 연결.

## I-8. 통합 생산 운영

**메뉴:** `운영` → **통합 생산 운영**

계획·지시·LOT·재고를 탭으로 처리.

| 요소 | 기능 |
|------|------|
| 탭 | 생산계획 / 작업지시 / 생산 LOT / 재고현황 |
| 요약 바 | 잔여배정, LOT배정 등 |
| 왼쪽 목록 | 품목·계획·지시 선택 |
| 지시 상태 **보류** | 현장 실적 **불가** |
| LOT 등록 불가 | 잔여배정 0 |
